# HRSkillsHub 设计文档

**日期：** 2026-05-13
**状态：** 已批准
**项目：** HRSkillsHub — 面向 HR 从业者的 AI Skills 分享平台

---

## 一、项目概述

HRSkillsHub 是一个专注于 HR 行业的 AI Skills 分享平台，对标 clawhub.ai 和 skillhub.cloud.tencent.com，但针对 HR 从业者的低技术门槛做了专项优化。

**核心差异点：**
- AI 工具无关：同时支持 Claude Code Skills 和通用提示词，兼容 DeepSeek、ChatGPT、Claude 等
- 零 GitHub 依赖：用户不需要 GitHub 账号即可使用和贡献内容
- 分级安装引导：按难度从低到高提供三种安装方式，附带使用说明

**目标用户：** HR 个人从业者（招聘专员、HRBP、薪酬专员等），技术水平较低，部分用户仅能访问 DeepSeek

## 二、技术架构

### 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端 & 后端 | Next.js 14（App Router）| 全栈一体，前后端一套代码 |
| 数据库 | PostgreSQL | 部署在同一台云服务器上 |
| 文件存储 | 阿里云 OSS 或腾讯云 COS | 存储用户上传的 ZIP 文件，按量计费 |
| 部署 | 阿里云 ECS 或腾讯云 CVM + Docker | 国内服务器，支持支付宝/微信付款 |
| 域名解析 | 阿里云/腾讯云域名解析 | 自购域名 |

### 部署架构

```
用户浏览器
    │
    ▼
自定义域名（阿里云/腾讯云解析）
    │
    ▼
云服务器（Docker 运行 Next.js 应用）
    ├── Next.js 应用
    ├── PostgreSQL 数据库（同服务器）
    └── 文件上传 → 阿里云 OSS / 腾讯云 COS
```

## 三、用户态页面设计

主域名下的所有用户可见页面。

### 页面列表

| 页面 | 路径 | 核心功能 |
|------|------|---------|
| 首页 | `/` | 搜索栏、精选/热门 Skills、分类入口 |
| 列表页 | `/skills` | 浏览、筛选、排序 |
| 详情页 | `/skills/[id]` | 三级安装引导、使用说明、评论、收藏 |
| 分享页 | `/submit` | 三种提交方式 |
| 用户中心 | `/profile` | 我的上传、收藏、个人资料 |

### Skill 详情页（核心页面）

详情页是平台最重要的页面，重点解决"看到了不知道怎么用"的问题。

**页面结构：**
```
┌─────────────────────────────────────────┐
│ Skill 名称                              │
│ 作者 · 分类 · 兼容 AI 标签 · 类型标签   │
│ 简介描述                                │
│ 下载数 / 收藏数（未登录可见）           │
├─────────────────────────────────────────┤
│ 如何使用（三种方式，按难度排列）        │
│                                         │
│ ① 简单｜复制提示词                     │
│   适合：所有 AI 用户（DeepSeek/ChatGPT）│
│   [复制提示词]（需登录）               │
│                                         │
│ ② 中级｜Claude Code 一键安装命令       │
│   适合：已安装 Claude Code 的用户      │
│   [复制安装命令]（需登录）             │
│   附：「什么是终端？」新手提示         │
│                                         │
│ ③ 进阶｜手动下载安装                   │
│   适合：想掌控文件位置的用户           │
│   [下载文件]（需登录）                 │
│   附：Windows / Mac 图文教程           │
├─────────────────────────────────────────┤
│ 使用说明 / 触发词 / 参数说明           │
│ 用户评论与评分（需登录才能发表）       │
└─────────────────────────────────────────┘
```

**安装按钮交互：**
- 点击「复制」后按钮变为「已复制 ✓」，3 秒后恢复
- 未登录用户点击任何操作按钮时，弹出登录提示框（不跳转页面）

### 操作权限

| 操作 | 未登录 | 已登录 |
|------|--------|--------|
| 浏览、搜索 | ✅ | ✅ |
| 查看下载数/收藏数 | ✅ | ✅ |
| 复制提示词（第一级） | ❌ 需登录 | ✅ |
| 复制安装命令（第二级） | ❌ 需登录 | ✅ |
| 下载文件（第三级） | ❌ 需登录 | ✅ |
| 收藏 | ❌ 需登录 | ✅ |
| 评论 | ❌ 需登录 | ✅ |
| 提交 Skill | ❌ 需登录 | ✅ |

**原则：未登录只能看，登录才能用。**

### 分享页（多种提交方式）

| 方式 | 适合人群 | 说明 |
|------|---------|------|
| 填写表单 | 所有用户 | 逐项填写名称、描述、内容、适用场景 |
| 在线编辑器 | 熟悉 Markdown 的用户 | 网页内 Markdown 编辑器，实时预览 |
| 上传 ZIP | 有完整 Skill 文件的用户 | 打包上传 + 填写一段使用说明 |

## 四、管理态页面设计

路径：`/admin`，仅管理员账号可访问，普通用户访问返回 403。

管理员账号不开放注册，由站长直接在数据库创建。

### 页面列表

| 页面 | 功能 |
|------|------|
| 审核队列 | 查看待审核提交，支持预览、通过、拒绝（附反馈意见） |
| Skills 管理 | 编辑、下架、修改分类/标签 |
| 用户管理 | 查看用户列表、设置认证贡献者、封禁账号 |
| 分类管理 | 增删改查 HR 分类标签 |

---

## 五、权限层级

```
普通用户       → 浏览、登录后可使用/评论/收藏/提交（首次需审核）
认证贡献者     → 提交后直接发布（无需审核）
管理员         → 完整后台权限
```

**认证贡献者晋升规则：** 首次提交审核通过后，由管理员手动在用户管理页设置。

## 六、数据模型

### Skills 表
```
id              唯一标识
title           标题
description     简介
content         完整内容（提示词文本 或 Skill 文件内容）
file_url        ZIP 文件地址（上传 zip 时使用）
type            类型：prompt（提示词）/ claude_skill（Claude Code Skill）
compatible_ai   兼容 AI：["deepseek", "chatgpt", "claude", "all"]
category        分类（招聘/绩效/薪酬/员工关系等）
install_count   使用/安装次数
favorite_count  收藏次数
download_count  下载次数
author_id       作者
status          状态：pending / published / rejected
created_at      提交时间
published_at    发布时间
```

### 用户表
```
id
email           邮箱（登录用，唯一）
nickname        昵称
role            角色：user / contributor / admin
avatar_url
created_at
```

### 收藏表
```
id
user_id
skill_id
created_at
```

### 下载记录表
```
id
skill_id
user_id
install_type    下载方式：copy_prompt / copy_command / download_file
created_at
```

### 评论表
```
id
skill_id
user_id
content
rating          评分 1-5
created_at
```

### 审核记录表
```
id
skill_id
admin_id        操作的管理员
action          approved / rejected
feedback        拒绝时的反馈意见
created_at
```

## 七、用户注册与登录

- **方式：** 仅支持邮箱注册登录（早期）
- **不依赖：** GitHub、微信、手机号（均需额外费用或不适合国内）
- **未来迭代：** 用户量增长后可考虑加入微信登录

---

## 八、筛选与排序

### 列表页筛选维度
- 分类（HR 细分领域：招聘、绩效、薪酬、员工关系、培训等）
- 类型（Claude Code Skill / 提示词）
- 兼容 AI（Claude / ChatGPT / DeepSeek / 通用）

### 排序方式
- 最新发布
- 最多下载
- 最多收藏
- 最高评分
